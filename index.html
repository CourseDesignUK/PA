Mesh Traversal The GLTFLoader iterates through the entire model hierarchy using the model.traverse() method. It specifically targets objects where child.isMesh is true. This ensures only actual geometry (the visible parts) is added to the components array for manipulation, ignoring helper objects like bones or cameras. 2. Radial Vector Calculation The direction each part moves is decided by its world position relative to the origin. JavaScript const worldPos = new THREE.Vector3(); child.getWorldPosition(worldPos); child.userData.explodeVec = worldPos.clone().normalize(); The code calculates a normalized vector (explodeVec) pointing from the center of the scene toward the center of that specific part. This creates a "radial explosion" effect where parts move outward from the center. 3. Translation Logic Within the animate() loop, the code updates the position of every mesh stored in the components array: JavaScript const shift = mesh.userData.explodeVec.clone().multiplyScalar(currentExplode); mesh.position.copy(mesh.userData.originalPos).add(shift); originalPos: The part's starting location. explodeVec: The direction determined in step 2. currentExplode: The distance multiplier (increased or decreased by the . and , keys). Gemini said In Blender, the distinction between a "Container" (Empty/Group) and a "Mesh Object" is fundamental. To ensure a part is recognized as a mesh and moved by your script, use these methods: 1. Join Multiple Meshes If you have parts that are currently separate but you want them to move as one "Mesh" unit: Select all the parts you want to combine. Press Ctrl + J. This merges them into a single Mesh Object with one name and one origin. 2. Convert "Primitives" or "Curves" to Mesh If your part is a Curve, Surface, Meta, or Text object, the script may not recognize it as a mesh. Select the object in the Viewport. Right-click and select Convert To > Mesh. The icon in the Outliner will change to the orange triangle, making it "visible" to child.isMesh. 3. Clear Parent (Separate from Containers) In your screenshot, many meshes are nested inside several "RootNode" containers. If you want a part to move independently based on its own position rather than its parent's position: Select the mesh object. Press Alt + P and select Clear and Keep Transformation. This moves the mesh to the top level of the hierarchy, ensuring its world position is calculated correctly by the script. 4. Set the Origin (Crucial for the "Explode" Logic) Your script moves parts based on their Origin (the orange dot). If the origin is at the bottom of the model (as seen in your screenshot), the part will move relative to that bottom point. Select the mesh. Right-click and select Set Origin > Origin to Geometry. The orange dot will move to the center of the part. Now, the explodeVec in your code will point directly away from the scene center from the center of that part, creating a much cleaner explosion effect. 5. Apply Transformations If you have moved or scaled the part in Object Mode, the "Original Position" might be recorded incorrectly in Three.js. Select the mesh. Press Ctrl + A. Select All Transforms.
